# Roadmap

This page describes the current documentation and project-direction priorities at a high level.

It should evolve over time as the project matures.

## Current documentation priorities

The current documentation direction strongly suggests a few immediate priorities:

- complete the narrative guide structure
- keep the QuickStart project aligned with the real public surface
- strengthen migrations coverage as a first-class docs area
- improve troubleshooting and FAQ coverage
- keep README, guide pages, examples, and API docs aligned

## Current product-direction themes

At a high level, the current project direction emphasizes:

- a lightweight, explicit ORM model
- strong readability around contexts, models, and datasets
- MySQL 8+ support
- migrations and schema tooling as a meaningful subsystem
- documentation that serves both beginners and advanced users

## Suggested near-term goals

Useful near-term goals include:

- finish and refine all guide pages
- add a richer feature-to-QuickStart mapping
- deepen migrations and schema examples
- improve DI and integration guidance
- expand troubleshooting coverage based on real user questions

## Suggested medium-term goals

Useful medium-term goals include:

- strengthen architecture notes
- clarify extension points and stability expectations
- add more operational guidance for migrations
- continue improving narrative consistency across docs layers

## Roadmap maintenance guidance

A roadmap page is most useful when it stays:

- honest
- lightweight
- regularly updated
- focused on priorities rather than promises

## Design guidance

A good default approach is:

- use the roadmap to communicate direction
- avoid treating it as a rigid guarantee
- keep it aligned with actual project and docs work
